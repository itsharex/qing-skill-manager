## 1. 产品概述

### 1.1 产品定位

Skills Manager 是一款专业的跨平台 AI Skills 管理桌面应用，帮助开发者统一管理 AI 开发环境中的 skills（技能插件）。

### 1.2 核心价值主张

- **统一仓库**: 将分散在不同 marketplace 的 skills 汇聚到本地统一管理
- **一键安装**: 通过符号链接快速将 skills 安装到目标 IDE
- **多 IDE 支持**: 支持 OpenCode、Claude Code、Cursor、VSCode 等多种 AI 开发环境
- **项目管理**: 按项目组织 skills，实现项目级 skill 配置

### 1.3 目标用户

- AI 辅助编程工具的使用者
- 需要在多个 IDE 间同步 skills 的开发者
- 需要管理大量自定义 skills 的高级用户

## 3. 用户故事 (User Stories)

### 3.1 本地 Skills 管理 (Local Skills)

#### 查看与搜索
- **US-001**: 作为用户，我希望查看系统中所有本地存储的 skills 列表，以便了解可用的 skills
- **US-002**: 作为用户，我希望按名称、描述或路径搜索/过滤本地 skills，以便快速找到特定 skill
- **US-003**: 作为用户，我希望看到本地 skills 的总数和过滤后的结果数，以便了解匹配数量

#### 选择与批量操作
- **US-004**: 作为用户，我希望使用复选框选择单个本地 skill，以便对特定 skills 执行操作
- **US-005**: 作为用户，我希望一次性选择所有可见 skills，以便高效执行批量操作
- **US-006**: 作为用户，我希望取消全选，以便重新开始选择

#### 安装 Skills
- **US-007**: 作为用户，我希望将单个本地 skill 安装到选定的 IDE 或项目，以便在该目标中使用 skill
- **US-008**: 作为用户，我希望一次性安装多个选中的本地 skills，以便高效设置多个 skills
- **US-009**: 作为用户，我希望在安装 skills 时选择目标 IDE 或项目，以便精确控制安装位置

#### 删除 Skills
- **US-010**: 作为用户，我希望删除单个本地 skill，以便从系统中移除不需要的 skills
- **US-011**: 作为用户，我希望一次性删除多个选中的本地 skills，以便高效清理多个 skills
- **US-012**: 作为用户，我希望一次性删除所有本地 skills，以便快速清空整个本地 skill 库
- **US-013**: 作为用户，我希望在删除 skills 前看到确认提示，以便确认我的破坏性操作

#### 目录与状态操作
- **US-014**: 作为用户，我希望在文件浏览器中打开 skill 的目录，以便直接访问其文件
- **US-015**: 作为用户，我希望看到每个 skill 被哪些 IDE 使用，以便了解 skill 的依赖关系
- **US-016**: 作为用户，我希望刷新/扫描本地 skills，以便更新列表反映任何变化

#### 导入与下载管理
- **US-017**: 作为用户，我希望从外部源导入 skills，以便向本地库添加新 skills
- **US-018**: 作为用户，我希望看到下载队列及其状态（待处理、下载中、完成、错误），以便监控下载进度
- **US-019**: 作为用户，我希望重试失败的下载，以便从错误中恢复
- **US-020**: 作为用户，我希望从下载队列中移除任务，以便清除不需要或卡住的任务

### 3.2 市场 (Marketplace)

#### 搜索与浏览
- **US-021**: 作为用户，我希望使用查询字符串在市场中搜索 skills，以便找到特定的 skills
- **US-022**: 作为用户，我希望按 Enter 键或点击按钮执行搜索，以便有明确的搜索提交方式
- **US-023**: 作为用户，我希望刷新市场结果，以便获取最新可用的 skills
- **US-024**: 作为用户，我希望加载更多结果（分页），以便浏览初始结果之外的 skills

#### 下载与更新 Skills
- **US-025**: 作为用户，我希望从市场下载 skills，以便将其添加到本地库
- **US-026**: 作为用户，我希望从市场更新现有的本地 skill，以便获取最新版本
- **US-027**: 作为用户，我希望看到 skill 的详细信息，包括名称、作者、星标数、安装数、描述和来源市场，以便在下载前评估 skills
- **US-028**: 作为用户，我希望看到哪些 skills 已在本地安装，以便知道应该下载还是更新
- **US-029**: 作为用户，我希望看到 skill 何时无法下载，以便了解无法下载的原因

#### 市场配置
- **US-030**: 作为用户，我希望配置市场设置，以便自定义使用哪些市场
- **US-031**: 作为用户，我希望启用/禁用单个市场源，以便控制搜索哪些市场
- **US-032**: 作为用户，我希望为需要认证的市场输入 API 密钥，以便访问付费或受限功能
- **US-033**: 作为用户，我希望切换 API 密钥的可见性，以便查看或隐藏我输入的密钥
- **US-034**: 作为用户，我希望看到市场状态（在线、需要密钥、不可用），以便了解哪些市场可用

## 4. 功能需求 (Functional Requirements)

### 4.1 核心功能模块

#### FR-001: Skill 发现与获取
**描述**: 系统必须支持从多个来源发现和获取 skills

**需求详情**:
- 支持从多个 marketplace 搜索 skills
- 支持按关键词搜索
- 支持分页加载结果
- 支持查看 skill 元数据（作者、星标、安装数、描述）
- 支持下载 skill 到本地仓库
- 支持更新已存在的本地 skill

**优先级**: P0 (核心功能)

---

#### FR-002: 本地 Skill 管理
**描述**: 系统必须提供完整的本地 skill 生命周期管理

**需求详情**:
- 扫描并显示本地仓库中的所有 skills
- 支持搜索和过滤本地 skills
- 支持批量选择和操作
- 支持删除单个或多个 skills
- 支持导入外部 skills
- 显示 skill 的使用状态（被哪些 IDE 使用）

**优先级**: P0 (核心功能)

---

#### FR-003: IDE 集成
**描述**: 系统必须支持将 skills 安装到多种 IDE 环境

**需求详情**:
- 支持预定义的 IDE 目录（OpenCode、Claude、Cursor 等）
- 支持添加自定义 IDE 配置
- 通过符号链接/目录连接将 skills 链接到 IDE
- 浏览 IDE 中已安装的 skills
- 区分管理的和未管理的 skills
- 支持从 IDE 卸载 skills

**优先级**: P0 (核心功能)

**技术约束**:
- Unix: 使用符号链接 (symlink)
- Windows: 优先使用目录连接 (junction)，回退到符号链接

---

#### FR-004: 项目管理
**描述**: 系统必须支持按项目组织 skills

**需求详情**:
- 创建和管理项目列表
- 为每个项目配置目标 IDE
- 自动检测项目中的 IDE 目录
- 将本地 skills 链接到项目
- 从项目导入 skills 到本地仓库
- 扫描项目中的现有 skills
- 处理导入冲突（保留/覆盖/共存）

**优先级**: P1 (重要功能)

---

#### FR-005: 市场聚合
**描述**: 系统必须支持从多个 marketplace 聚合 skills

**需求详情**:
- 支持 Claude Plugins marketplace
- 支持 SkillsLLM marketplace
- 支持 SkillsMP marketplace（需要 API 密钥）
- 显示每个市场的连接状态
- 支持启用/禁用特定市场
- 支持配置市场特定的 API 密钥
- 搜索结果去重（基于 sourceUrl）

**优先级**: P1 (重要功能)

---

#### FR-006: 更新管理
**描述**: 系统必须支持自动更新机制

**需求详情**:
- 启动时自动检查更新
- 手动检查更新
- 显示更新可用状态和版本号
- 下载更新并显示进度
- 安装更新并重启应用
- 显示当前版本信息

**优先级**: P2 (增强功能)

---

### 4.2 数据模型

#### FR-007: Skill 元数据
**描述**: 系统必须维护完整的 skill 元数据

**属性**:
```typescript
interface Skill {
  id: string;              // 唯一标识符
  name: string;            // 显示名称
  description: string;     // 描述
  path: string;            // 本地路径
  source: string;          // 来源（manager/link/local）
  author?: string;         // 作者
  namespace?: string;      // 命名空间
  marketId?: string;       // 来源市场
  stars?: number;          // GitHub 星标数
  installs?: number;       // 安装数
}
```

**优先级**: P0

---

#### FR-008: 项目配置
**描述**: 系统必须维护项目配置数据

**属性**:
```typescript
interface ProjectConfig {
  id: string;              // 唯一标识符
  name: string;            // 项目名称
  path: string;            // 项目路径
  ideTargets: string[];    // 目标 IDE 列表
  detectedIdeDirs: ProjectIdeDir[];  // 检测到的 IDE 目录
}
```

**优先级**: P1

---

#### FR-009: IDE 配置
**描述**: 系统必须维护 IDE 配置

**属性**:
```typescript
interface IdeOption {
  id: string;              // 唯一标识符
  label: string;           // 显示名称
  globalDir: string;       // 全局目录（相对或绝对路径）
  projectDir?: string;     // 项目级目录
}
```

**支持的 IDE**:
- OpenCode: `.config/opencode/skills`
- Claude Code: `.claude/skills`
- Cursor: `.cursor/skills`
- VSCode: `.github/skills`
- Windsurf: `.windsurf/skills`

### 4.3 文件系统操作

#### FR-010: Skill 存储
**描述**: 系统必须维护统一的 skill 存储位置

**需求详情**:
- 默认存储位置: `~/.skills-manager/skills/`
- 每个 skill 是一个包含 `SKILL.md` 文件的目录
- Skill 名称从 `SKILL.md` frontmatter 或目录名提取
- 支持目录名清理（移除 Windows 保留名、特殊字符）

**优先级**: P0

---

#### FR-011: 符号链接管理
**描述**: 系统必须安全地创建和管理符号链接

**需求详情**:
- 创建从 IDE 目录到 manager 目录的符号链接
- 验证目标路径在 home 目录内（安全约束）
- 处理已存在的链接（跳过如果已链接到同一目标）
- 支持跨文件系统链接（Windows junction）
- 验证链接目标有效性

**优先级**: P0

**安全约束**:
- 禁止路径遍历攻击 (`..`)
- 禁止绝对路径指向系统目录 (`/etc`, `/sys`, `/proc`, `/dev`, `/root`)
- 禁止 Windows 保留名称 (`CON`, `PRN`, `AUX`, `NUL`, `COM1-9`, `LPT1-9`)

---

#### FR-012: Skill 导入
**描述**: 系统必须支持从外部源导入 skills

**需求详情**:
- 验证源目录包含 `SKILL.md`
- 复制目录内容到 manager 存储
- 处理命名冲突（如果目标已存在则报错）
- 保留原始目录结构

**优先级**: P1

---

#### FR-013: Skill 纳管
**描述**: 系统必须支持将 IDE 中的 skills 纳入管理

**需求详情**:
- 将 IDE 中的 skill 复制到 manager 存储
- 创建从原位置到 manager 的符号链接
- 如果链接失败则回退到本地副本
- 验证源目录包含 `SKILL.md`

**优先级**: P1

## 7. 接口需求

### 7.1 用户界面

#### UI-001: 导航结构
```
主标签页:
├── 已有 Skills (Local)
├── Market (市场)
├── IDE 浏览 (IDE)
├── 项目管理 (Projects)
└── 设置 (Settings)
```

#### UI-002: 主要组件
- **Panels**: LocalPanel, MarketPanel, IdePanel, ProjectsPanel, SettingsPanel
- **Modals**: InstallModal, UninstallModal, ProjectAddModal, ProjectConfigModal, ConflictResolutionModal
- **Overlays**: LoadingOverlay, Toast
- **Utilities**: DownloadQueue, UpdateChecker

#### UI-003: IDE 浏览页面 (IdePanel)

**页面功能**:
显示和管理各 IDE 目录中的 skills，支持查看、纳管、卸载操作。

**显示内容**:

| 元素 | 说明 |
|------|------|
| **IDE 筛选器** | 按钮组，切换当前查看的 IDE（默认只有 OpenCode） |
| **自定义 IDE 添加** | 输入名称和路径，添加新的 IDE 配置 |
| **Skill 卡片** | 显示 skill 名称、来源类型、管理状态、完整路径 |
| **操作按钮** | 打开目录、纳管（仅未托管）、卸载 |
| **批量操作** | 纳管选中、卸载选中 |

**关键概念**:

**托管 (managed)**:
- 通过 Skills Manager 安装
- 实际存储在 `~/.skills-manager/skills/`
- IDE 目录中是符号链接
- 可以被统一管理（更新、卸载）

**未托管 (unmanaged)**:
- 用户手动复制到 IDE 目录
- 不在 Skills Manager 控制下
- 显示橙色边框标记
- 需要先"纳管"才能统一管理

**来源类型**:
- **链接 (link)**: skill 是符号链接，实际存储在 manager 目录
- **本地 (local)**: skill 是实际目录，不是链接

**注意**: 当前版本默认只配置 OpenCode，其他 IDE 需手动添加或在 `constants.ts` 中扩展 `defaultIdeOptions`。

---

#### UI-004: Skill 安装流程

**安装到全局 IDE**:

1. 在 **Local Skills** 页面选中 skills
2. 点击"批量安装到编辑器"
3. `InstallModal` 弹出，显示两列：
   - **左列（全局 IDE）**: 安装到 IDE 全局配置目录
   - **右列（项目）**: 安装到特定项目的 IDE 目录
4. 选择全局 IDE，点击"安装到 IDE"
5. 调用 `link_local_skill` 命令创建符号链接

**项目级 Skill 转成 IDE 全局**:

**方法 1: 纳管 + 重新安装**
1. 在 **IDE 浏览**页面找到项目中的 skill（显示为"未托管"）
2. 点击"纳管"按钮，执行 `adopt_ide_skill`:
   - 将 skill 从项目目录**复制**到 `~/.skills-manager/skills/`
   - 在项目目录**创建符号链接**替换原目录
3. 在 **Local Skills** 页面选中刚纳管的 skill
4. 点击"安装到编辑器"，选择全局 IDE

**方法 2: 从项目导入**
1. 在 **项目管理**页面选择项目
2. 点击"从项目导出" / "扫描项目中的 Skills"
3. 选择要导入的 skills，解决冲突（如果有）
4. 执行 `import_local_skill` 复制到 `~/.skills-manager/skills/`
5. 回到 **Local Skills** 页面，安装到全局 IDE

**安装目标对比**:

| 安装目标 | 存储位置 | 适用范围 | 关键命令 |
|---------|---------|---------|----------|
| **全局 IDE** | `~/.config/opencode/skills/` (symlink) | 所有项目 | `link_local_skill` |
| **项目级** | `~/project/.config/opencode/skills/` (symlink) | 仅该项目 | `link_local_skill` |
| **纳管** | 复制到 `~/.skills-manager/`，原位置变 symlink | - | `adopt_ide_skill` |
| **导入** | 复制到 `~/.skills-manager/skills/` | - | `import_local_skill` |
