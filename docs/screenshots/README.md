# Screenshot Refresh Guide

The inherited screenshots from the upstream project were removed during the Qing Skill Manager rebrand.

## Why they were removed

- They still reflected the old product branding
- They could confuse users about whether this repository is an independent fork
- The project UI changed significantly around project management, versioning, and clone/copy workflow

## Recommended new screenshot set

Capture these pages after launching the latest app build:

1. `local-overview.png`
   - Local Skills page with Qing Skill Manager branding visible
2. `market-search.png`
   - Market page with a populated search result list
3. `ide-browser.png`
   - IDE Browser page showing managed copies
4. `projects-panel.png`
   - Project management page showing project snapshots and managed version state
5. `version-manager.png`
   - Version management modal with default strategy and imported versions

## Capture recommendations

- Use consistent window size (for example 1440×960)
- Prefer zh-CN for the current website/README narrative unless you plan dual-language assets
- Ensure the app name shown is **Qing Skill Manager**
- Avoid showing inherited repository names or legacy screenshot assets
